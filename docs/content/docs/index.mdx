OpenNexus is a Rust CLI package (`opennexus`) centered on the `opennexus` command. It installs governance and workflow assets into your project, keeps command specs in sync, and gives teams a repeatable way to run context-driven development.

## Start Here

1. Install the CLI:

```bash
cargo install opennexus
```

Alternative install channels:

| Ecosystem | Install Command |
|-----------|------------------|
| npm | `npm install -g opennexus` |
| uv / PyPI | `uv tool install opennexus` |
| Homebrew | `brew tap Alpha-Innovation-Labs/tap && brew install opennexus` |
| Scoop | `scoop bucket add alpha-innovation-labs https://github.com/Alpha-Innovation-Labs/scoop-bucket && scoop install opennexus` |
| AUR | In progress (not published yet) |

2. Initialize OpenNexus in your project root:

```bash
opennexus setup
```

## Core Commands

| Command | What it does |
|---------|---------------|
| `opennexus setup` | Installs/refreshes `.nexus` assets and `.opencode/command` links in the current repo |
| `opennexus update` | Upgrades the installed `opennexus` command to the latest published version |
| `opennexus uninstall` | Removes the globally installed `opennexus` command |
| `opennexus --help` | Shows available commands and options |

## Harness Structure

OpenNexus ships context governance assets in a feature-first layout under `.nexus/context/<project>/<feature>/`.

- Numbered context files (`PRJ_NNN-*.md`) define desired outcomes and E2E-observable next actions.
- `index.md` files at project and feature levels store operational knowledge.
- Context IDs are project-scoped and increment across all feature folders in that project.

## Features

### Slash Commands

OpenCode command-form workflows are documented in the [Slash Commands catalog](/docs/commands).

| Slash Command | Purpose |
|---------------|---------|
| `nexus` | Opens context selection workflow (fuzzy context picker) |
| `nexus-context-create` | Scans for overlap, splits distinct outcomes, and creates context specs with project-scoped IDs |
| `nexus-context-sync` | Syncs context updates from completed implementation work |
| `nexus-context-review` | Audits context quality and coverage before execution |

### Skills

OpenNexus ships built-in skills under [Skills -> Rust](/docs/skills/rust).

#### Rust

| Skill | Purpose |
|-------|---------|
| [`ratkit`](/docs/skills/rust/ratkit) | Operational guide for the ratkit Rust TUI ecosystem, feature flags, modules, and usage patterns |
| [`opencode-rs-sdk`](/docs/skills/rust/opencode-rs-sdk) | Operational guide for OpenCode Rust SDK HTTP, SSE streaming, and managed server/client workflows |

### llms.txt Generation

OpenNexus can use AIs to generate a friendly `llms.txt` file that captures project knowledge in an agent-consumable format.

- Generated `llms.txt` content can be converted into reusable skills.
- Built-in skill artifacts are stored under `.nexus/ai_harness/skills/`.
- Use the slash-command workflow to iterate from project context to `llms.txt`, then from `llms.txt` to skill artifacts.
