description: Navbar, hero, and layout contracts used in the docs app

## Navbar

- Sticky header with clear active route highlighting.
- Desktop links with mobile navigation fallback.
- Repository access and theme controls remain reachable at all sizes.

## Hero

- Two-column desktop layout and one-column mobile layout.
- Quickstart command with copy interaction and short success feedback.
- Tabbed example panel with instant content switching.

## Internal Docs Layout

- Left navigation tree from Fumadocs content source.
- Right-side TOC generated from page headings.
- Heading anchors support deep links and in-page traversal.
